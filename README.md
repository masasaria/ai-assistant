Meeting AI Assistant

A local AI assistant that listens to your meeting audio in real time, transcribes questions using Whisper, and answers them based on a document you upload — designed for thesis defenses, technical interviews, and presentations.


How it works


Upload your .docx or .pdf document (thesis, report, technical spec, etc.)
The app processes it and builds a knowledge base
Start listening — the app transcribes audio and detects questions automatically
Answers stream back in real time, grounded exclusively in your document


Audio is transcribed locally using faster-whisper. Answers are generated by either a local Ollama model or the Anthropic API (Claude). No audio or document content is sent anywhere unless you choose the Anthropic API option.


Requirements


Python 3.10 or higher — python.org/downloads
During installation, check "Add Python to PATH"
Windows 10 / 11 (64-bit) — tested on Windows; should work on macOS/Linux with minor adjustments
A modern browser (Chrome recommended for system audio capture)



Installation

Open a terminal (Command Prompt or PowerShell) and run:

pip install flask ollama faster-whisper python-docx pdfplumber anthropic numpy SpeechRecognition pymupdf

This may take 2–5 minutes depending on your connection.


Answer engine — choose one

Option A — Local model with Ollama (offline, free)


Download and install Ollama from ollama.com
Pull a model:


   ollama pull gemma4:12b

(approx. 8 GB download — only needed once)
3. Start the Ollama server:

   ollama serve

Keep this window open while using the app.

Option B — Anthropic API / Claude (recommended for best quality)


Create an account at console.anthropic.com
Go to API Keys and generate a key (starts with sk-ant-api…)
No extra installation needed — paste the key directly in the app.


Recommended model: claude-sonnet-4-6 — significantly better at following complex instructions, staying in persona, and citing sources correctly compared to smaller models.


Running the app


Place meeting_assistant_v3_app.py in any folder
Open a terminal in that folder and run:


   python meeting_assistant_v3_app.py


The browser opens automatically at http://localhost:5000



Usage

Step 1 — Configure


Select Local (Ollama) or Anthropic API
If using Anthropic, paste your API key and choose a model
Select transcription engine: Whisper (local, more accurate) or Google (faster, requires internet)
Select audio source: Mic for your microphone, System to capture VM/meeting audio


Step 2 — Load document


Click Browse… and upload your .docx or .pdf
Wait for processing to complete (1–2 minutes the first time)


Step 3 — Listen


Select the language
Click Start Listening
For system audio: select your screen in the Chrome share dialog and tick "Share system audio"
Questions detected in the audio are automatically transcribed and answered



Capturing meeting audio from a virtual machine

If your meeting runs inside a VM (VMware, VirtualBox, etc.) and you want the app to capture that audio on the host:


In the app, switch audio source to 🖥️ System
Click Start Listening
In Chrome's share dialog, select the screen where the VM is running and enable "Share system audio"


The app captures the host's system audio output, which includes whatever the VM is playing through the host speakers.

Notes


Document processing runs once per session. After that, answers arrive in seconds.
Answers are based exclusively on the uploaded document — the model will not use outside knowledge.
Word documents using the built-in citation manager (IEEE, APA, etc.) are fully supported — author names are extracted from the citation XML and injected into the context automatically.
The Anthropic API sends document excerpts (not the full document) to Claude's servers to generate answers. If confidentiality is a concern, use Ollama instead.
